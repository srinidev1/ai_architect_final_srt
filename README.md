# Chatbot Application with RAG, MCP Tools, and Evaluation Framework

## Overview

This project is a chatbot application built with multiple agents and orchestration using LangGraph. It supports:

- **Delaware Annual Report Filing Search** using RAG (Retrieval-Augmented Generation)
- **Weather Information** via MCP tools
- **Natural Disaster Event Search** using LLM to convert natural language into Pandas queries and execute them via MCP tools on CSV data

The chatbot accepts user questions through a Streamlit UI and returns responses generated by LLMs.

---

## Features

### 1. Filing Search (RAG)
- Searches Delaware annual report filings using a RAG pipeline.
- Retrieves relevant filing context from a vector store.
- Uses LLM to generate answers based on retrieved context.

### 2. Weather Search
- Uses MCP weather tools to fetch weather information.
- Returns weather responses through LLM generation.

### 3. Disaster Event Search
- Supports querying natural disaster events stored in a CSV dataset.
- Converts natural language questions into Pandas queries using an LLM.
- Uses an MCP tool to execute the generated Pandas query against the CSV file.
- Uses LLM to summarize tool results and generate the final response.

### 4. Generic Agent
- Returns a generic response for questions outside of filings, weather, and disaster events

### 5. Security
- Orchestration includes secured system prompts to mitigate Prompt injection attacks
- Indirect prompt injection from retrieved RAG context is mitigated when sending context to LLM for generating response

## Models and Frameworks

- **Orchestration model:** `gemini-2.5-flash-lite` via DIAL API
- **Response generation model:** `gpt-4` via DIAL API
- **Response evaluation/judging model:** `gemini-2.5-flash` via DIAL API

Model configuration is defined in:

```python
utils/models.py
```

Frameworks and libraries used:
- LangGraph for orchestration workflow
- Streamlit for UI
- Pandas for CSV querying and data processing
- KaggleHub for dataset download and caching

---

## Architecture

- **LangGraph** is used to build the orchestration workflow.

- `supervisor.py` routes user requests to the appropriate agent.

- Available agents:
  - **Filing Agent** → invokes `filing_service.py`
  - **Weather Agent** → invokes `weather_service.py`
  - **Disaster Event Agent** → invokes `disaster_event_service.py`
  - **General Agent** → handles generic questions

- Service responsibilities:
  - **Filing Service**
    - Retrieves relevant context from vector store
    - Invokes LLM for response generation

  - **Weather Service**
    - Calls weather MCP tools
    - Uses tool output to generate final response

  - **Disaster Event Service**
    - Converts natural language to Pandas query using LLM
    - Calls disaster event MCP tool to execute query against CSV
    - Uses tool output to generate summarized response

## Project Structure

- **agents/**  
  Contains all chatbot agents:
  - Filing Agent
  - Weather Agent
  - Disaster Event Agent
  - General Agent

- **orchestration/**  
  Contains LangGraph orchestration flow and routing logic.  
  `supervisor.py` is responsible for invoking the appropriate agent based on user query.

- **services/**  
  Contains service layer implementations used by agents:
  - `filing_service.py`
  - `weather_service.py`
  - `disaster_event_service.py`

- **mcp_server/**  
  MCP server implementation and tool registration.  
  Server entry point:
  ```bash
  python mcp_server/server.py
  ```

- **mcp_server/tools/**  
  Contains MCP tools:
  - Weather tools
  - Disaster event search tool

- **mcp_client/**  
  Contains MCP client-related code including:
  - async runner
  - tool executor
  - MCP client initialization

- **data/**  
  Stores:
  - RAG knowledge base
  - vector store

  To rebuild vector store:
  ```bash
  python scripts/build_vector_store.py
  ```

- **natural_disasters/**  
  Stores downloaded disaster events CSV file.

  Dataset is downloaded once using `kagglehub` from:
  `disaster_event_service.py -> download_disaster_events()`

- **tests/**  
  Contains evaluation modules:
  - `classifiereval/` for classifier testing
  - `responseeval/` for final response evaluation

- **views/**  
  Streamlit UI rendering files for:
  - Chat view
  - Evaluation view

- **utils/**  
  Shared utilities including model configuration in:
  ```python
  utils/models.py
  ```

- **main.py**  
  Streamlit application entry point.
